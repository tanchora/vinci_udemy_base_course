Репрезентации Vinci-модели

Работа с Nasdanika Vinci заключается в том, чтобы:

  1. Создать файл модели (мы научимся делать это в следующем уроке);
  2. Открыть модель в виде диаграммы или в виде дерева элементов.
  3. Нарисовать структуру сайта: создать элементы (страницы), задать связи между страницами и разделами сайта.
  4. Добавить к элементом модели (= страницам сайта) контент и настройки в дереве элементов модели.

Таким образом, для модели существуют два способа представления или две репрезентации. Давайте их откроем в области model explorer. Выделяем файл модели. Мы его можем узнать по характерной иконке с витрувианским человечком и по расширению .vinci.

Рядом с этим файлом есть стрелочка показывающая, что его можно развернуть и посмотреть его структуру. Разворачиваем и видим корневой элемент модели. Это тот элемент, из которого будет расти вся наша модель как дерево.

Для того, чтобы открыть диаграмму или дерево элементов нам необходимо создать репрезентацию (представление, внешнее проявление модели). Это делается из контекстного меню корневого элемента:

Как мы видим, программа предлагает нам создать новую Action Diagram и новое Vinci Adapter Factory Tree. Давайте для простоты первую репрезентацию будем называть диаграммой, а вторую репрезентацию будем называть деревом элементов.

После создания репрезентации открываются большом левом окне. Это окно редактирования и наши репрезентации существуют в нем в виде вкладочек, но мы можем их разделить на отдельные окна.
